Process the command line arguments and return a tuple consisting of the user options structure, accelerate-examples options (including options for criterion and test-framework), and a list of unrecognised command line arguments.

Since criterion and test-framework both bail if they encounter unrecognised options, we run getOpt' ourselves. This means that the error messages might be slightly different.

Any command line arguments following a "--" are not processed, but are included as part of the unprocessed arguments returned on output.
